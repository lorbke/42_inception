====================MariaDB====================
- upon installation, mariaDB creates mysql user
- creating MariaDB folders
    - mkdir -p /var/lib/mysql (for storing data)
    - mkdir -p /run/mysqld (for communication)
    - chown -R mysql:mysql /run/mysqld
    - chown -R mysql:mysql /var/lib/mysql
- initializing database
    - mysql_install_db --user=mysql --datadir=/var/lib/mysql
- copy scripts into container
    - COPY conf/edit_server_config.sh .
    - COPY conf/create_wordpress_db.sh .
- start MariaDB daemon
    - mysqld_safe --user=mysql

====================Nginx====================

    - openssl genrsa -out lorbke.42.fr.key 2048 (generate 2048 bit private key)
    - openssl req -new -key lorbke.42.fr.key -out lorbke.42.fr.csr (generate certificate signing request)
    - fill out shit

===================General===================
- my SSL cert is not issued by a trusted CA and therefore produces a warning in Chrome
- nginx runs on user "nginx" by default
- docker exec -it [container name] bash (can be used to go into container during runtime)
- nginx is essentially the middleware between clients and php/wordpress and is helping to balance the load etc.
- COPY directive doesn't work on docker volumes, because file is COPYed first by dockerfile, then volume is mounted and whole folder structure is now routing to a different path
